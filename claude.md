# CLAUDE.md - Team Matching Application

## Project Overview

แอปพลิเคชันจับกลุ่มทีม (Team Matching App) สำหรับใช้ในสถานศึกษา ช่วยให้อาจารย์สร้างห้องและจับกลุ่มนักเรียนโดยอิงจากการวิเคราะห์ MBTI จาก templates คำถามต่างๆ ระบบจะจัดกลุ่มให้สมาชิกมีจุดเด่นที่หลากหลายและไม่ซ้ำกัน

---

## User Roles

- **Host (อาจารย์)** — สร้างห้อง, กำหนดค่า, กด match จับกลุ่ม
- **User (นักเรียน)** — เข้าร่วมห้อง, ตอบคำถาม, รอจับกลุ่ม

---

## Authentication & Onboarding

1. ผู้ใช้สมัครสมาชิกและ login เข้าสู่ระบบ
2. เมื่อเข้ามาครั้งแรก ต้องกรอก **ชื่อ** และ **ตั้งอวาตาร์** (เพื่อให้เพื่อนร่วมทีมจำได้)
3. เลือกสถานะ: **User (นักเรียน)** หรือ **Host (อาจารย์)**

---

## หน้าหลัก (Home)

หน้าหลักเหมือนกันทั้ง User และ Host ประกอบด้วย 5 ปุ่มหลัก:

| ปุ่ม | หน้าที่ |
|------|---------|
| **Create** | สร้างห้องเพื่อให้นักเรียนเข้า join |
| **Join** | กรอก Room ID (6 หลัก) เพื่อเข้าห้อง |
| **Team** | เข้าดูห้องที่เคยจับกลุ่มไว้แล้ว |
| **Templates** | เลือก templates คำถาม MBTI ตามหมวดงาน (เช่น Programming) |
| **My Type** | แสดงผล MBTI type ที่ได้จากแต่ละ template |

---

## Flow: Create Room (Host)

### ขั้นตอนที่ 1 — ตั้งค่าห้อง
- กรอก **ชื่อห้อง**
- กรอก **รายละเอียด** (เช่น จับกลุ่มเพื่อทำงานอะไร, กำหนดส่งงาน)

### ขั้นตอนที่ 2 — กำหนดจำนวน
- ตั้ง **limit จำนวนคนสูงสุด** ที่เข้าห้องได้
- กำหนด **จำนวนคนต่อกลุ่ม**

### ขั้นตอนที่ 3 — กำหนด Type Composition
- เลือกว่าสมาชิกในกลุ่มจะประกอบด้วย **type อะไรบ้าง**

### ผลลัพธ์
- ระบบสร้าง **Room ID** ให้ host
- Host คัดลอก Room ID ส่งให้นักเรียน

---

## Flow: Join Room (User)

1. กดปุ่ม **Join** ที่หน้าหลัก
2. กรอก **Room ID 6 หลัก** แล้วกดถัดไป
3. ระบบแสดงรายละเอียดห้อง — นักเรียนตรวจสอบว่าใช่ห้องที่ต้องการ
4. กด **Join** เพื่อยืนยันเข้าห้อง
5. เห็นรายละเอียดห้อง + จำนวนสมาชิกที่เข้ามาแล้ว
6. กดปุ่ม **Ready** เพื่อยืนยันว่าพร้อมจับกลุ่ม
7. สถานะเปลี่ยนเป็น **"รอ"** จนกว่า Host จะกด match

---

## Flow: Match (Host)

1. Host เห็นจำนวนคนที่เข้าห้องและสถานะ Ready ของแต่ละคน
2. เมื่อนักเรียน Ready ครบ → Host กดปุ่ม **Match**
3. ระบบจับกลุ่มตามค่าที่กำหนดไว้
4. Host เห็นผลทั้งหมด: สมาชิกแต่ละกลุ่ม + แท็กบทบาท + หัวหน้ากลุ่ม

---

## หลักการจับกลุ่ม (Matching Algorithm)

- นำคำตอบทั้ง **10 ข้อ** ของนักเรียนมาวิเคราะห์
- วิเคราะห์ว่านักเรียนแต่ละคนเหมาะกับ **ตำแหน่งงานไหน** และมี **จุดเด่นอะไร**
- จับกลุ่มให้ **คละกัน** — แต่ละกลุ่มมีสมาชิกที่มีจุดเด่นไม่ซ้ำกัน
- สมาชิกแต่ละคนจะมี **แท็ก (tag)** แสดงความถนัด

---

## ผลลัพธ์หลัง Match (User)

- เห็นรายชื่อสมาชิกในกลุ่ม
- แต่ละคนมี **แท็ก** บอกจุดเด่น/ความถนัด
- กดปุ่ม **ⓘ** เพื่ออ่านความหมายของแต่ละแท็ก
- สามารถ **แก้ไขชื่อทีม** ได้

---

## การเลือกผู้นำทีม (Leader Selection)

มี 2 โหมด:

### โหมด 1: Vote
- สมาชิกเลือกคนที่คิดว่าเหมาะเป็นผู้นำ แล้วกด **Vote**

### โหมด 2: Analyze
- ระบบนำคำตอบของสมาชิกทุกคนในกลุ่มมาวิเคราะห์
- เลือกคนที่มีแนวโน้มเป็นผู้นำมากที่สุด
- คนที่ถูกเลือกจะได้รับ **แท็กพิเศษ (ไอคอนมงกุฎ 👑)** แสดงว่าเป็นหัวหน้าทีม
- สมาชิกในทีมกด **ยืนยัน** หัวหน้าทีม

## Mini Chat

- แต่ละทีมมี **มินิแชท** ในตัว
- ใช้ทักทาย หรือนัดหมายสถานที่ทำงานกัน

---

## Session Recovery

- หากนักเรียนปิดแอปโดยไม่ตั้งใจ
- กลับมาหน้าหลัก → กดปุ่ม **Team**
- ระบบแสดงทีมที่เคยจับกลุ่มไว้แล้ว

---

## Templates System

- มี templates คำถามหลายหมวด (เช่น Programming)
- แต่ละ template มีชุดคำถาม MBTI เฉพาะหมวด
- ผลวิเคราะห์บอกว่าผู้ใช้เป็น **MBTI type ไหน** ในบริบทนั้นๆ
- ผลนี้ใช้ในการจับกลุ่มและกำหนดตำแหน่งที่เหมาะสม

---

## หน้า My Type

- แสดง MBTI type ที่ได้จากแต่ละ template ที่เคยทำ
- ผู้ใช้เห็นภาพรวมว่าตนเองมีบุคลิกแบบไหนในแต่ละบริบทงาน
